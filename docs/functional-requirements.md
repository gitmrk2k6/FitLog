## 2. 機能詳細（機能定義）

### F-01 ユーザー認証機能

**ユーザー登録**

- 入力: ユーザー名（表示名）、メールアドレス、パスワード
- バリデーション:
  - メールアドレス形式チェック・一意（重複登録不可）
  - ユーザー名は一意（重複登録不可）
  - パスワードは8文字以上、英字・数字を両方含む
- パスワードは Bcrypt でハッシュ化して保存
- 登録成功後、ログイン画面へ遷移

**ログイン / ログアウト**

- ログイン: メールアドレス＋パスワード。成功で JWT 発行、ダッシュボードへ遷移
- 認証失敗: 「メールアドレスまたはパスワードが正しくありません」を表示
- 未ログインで保護機能を操作した場合はログイン画面へリダイレクト
- ログアウト: クライアント側で JWT を破棄

---

### F-02 トレーニング記録機能

**記録作成**

- ログイン済みユーザーのみ。1記録 = 「実施日 + 1件以上の `種目 × セット`」
- 入力:
  - 実施日（必須、デフォルト当日）
  - 種目（必須、種目マスタから選択。マスタにない場合は新規追加可）
  - セット（必須、1件以上）: 重量(kg, 0以上の数値) × 回数(reps, 1以上の整数)
  - メモ（任意、最大280文字）
  - 写真（任意、F-10）
- 保存時に F-09（自己ベスト判定）を実行

**記録編集 / 削除**

- 自分の記録のみ編集/削除可能
- 削除確認ダイアログを表示してから実行
- 記録削除時に紐づくセット・ナイストレ・アドバイスも削除（CASCADE）

---

### F-03 記録一覧/詳細機能

**一覧表示**

- 自分の記録を実施日降順で表示
- 各記録に表示する情報:
  - 実施日
  - 種目数・総セット数
  - 総ボリューム（Σ 重量 × 回数）
  - ナイストレ数・アドバイス数
  - 写真サムネイル（存在する場合）

**詳細表示**

- 種目ごとのセット内訳（重量×回数）
- メモ・写真
- 自己ベスト更新があった種目にバッジ表示（F-09）
- ナイストレ・アドバイス（F-04 / F-05）

---

### F-04 ナイストレーニング機能

- ログイン済みユーザーが他ユーザーの記録に「ナイストレ」を付与（投稿者本人以外）
- 同一ユーザーは同一記録に1回のみ（重複不可）、解除可能
- 一覧/詳細でナイストレ数を表示。自分が付与済みかをアイコンで識別

---

### F-05 アドバイス・応援機能

**投稿**

- ログイン済みの任意ユーザーが他ユーザーの記録にコメント可能
- 入力: テキスト（必須、最大140文字）

**一覧 / 削除**

- 記録に紐づくアドバイスを投稿日時昇順で表示（コメント者名・本文・日時）
- 自分のアドバイスのみ削除可能（確認ダイアログ後に実行）
- 一覧/詳細でアドバイス数を表示

---

### F-06 トレーニング仲間フォロー機能

- ユーザー名（部分一致）で検索。検索結果にフォロー状態を表示
- フォロー/アンフォロー（自分自身不可・重複不可）
- フォロー中ユーザーの記録を集約した「フォロー中フィード」を表示
- プロフィールにフォロー中数・フォロワー数、各一覧を表示

---

### F-07 目標設定＋達成率機能

**目標設定**

- 入力: 期間種別 `period_type`（weekly / monthly）、指標 `metric`（sessions=実施回数 / volume=総ボリューム）、目標値 `target_value`（正の数）
- 同一 period_type の有効な目標は1件（更新可能）

**達成率表示**

- 現在期間（今週/今月）の実績を集計し、達成率 = 実績 / 目標値（上限100%表示・超過は達成扱い）
- ダッシュボードに達成率リング、集計画面に週次/月次の達成率推移グラフ
- 集計は PostgreSQL の `date_trunc` で期間バケットを生成して算出

---

### F-08 ストリーク＋ヒートマップ機能

**ストリーク**

- 記録のある実施日をもとに「現在の連続記録日数」を算出（当日または前日まで連続していれば継続）
- 最長ストリークも保持/算出

**カレンダーヒートマップ**

- 直近 N か月（例: 5か月）の日別記録量を GitHub 風グリッドで濃淡表示
- `generate_series` で日付軸を生成し、日別の記録有無/ボリュームを LEFT JOIN して欠損日も表示

---

### F-09 自己ベスト(PR)自動検出機能

筋トレ実施者が知りたい2軸を **分けて** 管理・表示する。

1. **最大重量**: 回数に関係なく挙上した最大重量（「1回でも最大何kgを扱えたか」）。達成時の回数・日付も保持
2. **推定1RMベスト**: 推定1RM = weight × (1 + reps / 30)（Epley式）が最大のセット（「重量×レップでの強さ」）。元の重量・回数・日付も保持

- F-02 の記録保存時、保存セットが①または②の基準値を超えた場合に、超えた指標を特定して「自己ベスト更新」と判定（どちらを更新したか区別する）
- ダッシュボードに種目別の①②を併記表示。記録詳細では更新した指標のバッジを表示
- 推定1RMは Epley 式。10レップ超は精度が落ちるため「参考値」と注記
- 有酸素・自重種目（重量0）は重量/1RM指標が機能しないため本機能の対象外（将来: 最大回数・距離・タイムで別途PR管理）
- 基準値は `personal_records` に保持、または記録から算出（実装方針は database-design.md 参照）

---

### F-10 写真の保存機能

**アップロード**

- トレーニング記録に写真を1枚添付可能（任意）
- 対応形式: JPEG, PNG。最大ファイルサイズ: 5MB
- バリデーション: MIME type チェック、サイズチェック（超過時エラー）

**保存 / 表示**

- アップロード画像は AWS S3 に保存（教材 RaiseTimeLine の S3 構成を流用）
- DB には画像URLのみ保持
- 記録一覧（サムネイル）・詳細で表示。写真なしでもエラーにならない

---

## 3. ユースケース

### UC-01: 新規ユーザーが登録して記録を始める

1. 新規登録画面でユーザー名・メール・パスワードを入力（F-01）
2. ログイン後、記録入力画面で実施日・種目・セットを入力して保存（F-02）
3. ダッシュボードにストリーク・達成率が反映される（F-07, F-08）

### UC-02: 記録して自己ベストを更新する

1. ログイン済みユーザーがベンチプレスのセットを記録（F-02）
2. 過去最大重量を超えたため「自己ベスト更新」バッジが表示される（F-09）
3. ダッシュボードの「直近のPR更新」に反映される（F-09）

### UC-03: 仲間の記録を見て応援する

1. ユーザー検索で仲間を探しフォローする（F-06）
2. フォロー中フィードで仲間の記録を閲覧（F-06, F-03）
3. 「ナイストレ」を付け、アドバイスをコメントする（F-04, F-05）

### UC-04: 目標を設定して達成率を追う

1. 目標設定画面で「週3回」を設定（F-07）
2. 週内に記録を重ねるとダッシュボードの達成率リングが上昇（F-07）
3. 集計画面で週次の達成率推移を確認（F-07）

### UC-05: 継続状況を振り返る

1. ダッシュボードでカレンダーヒートマップを確認（F-08）
2. 連続記録日数（ストリーク）を確認しモチベーションを維持（F-08）
3. 集計画面で月間総ボリュームの推移を確認（F-03, F-07）

### UC-06: 自分の記録を編集・削除する

1. 記録一覧から対象記録の詳細を開く（F-03）
2. セットの重量を修正して保存、または削除確認後に削除（F-02）
3. 削除時は紐づくセット・ナイストレ・アドバイスも削除される（F-02, F-04, F-05）
